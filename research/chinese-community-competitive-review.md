## 1. Aider 在掘金的传播模式

### 1.1 高互动文章类型

| 文章类型 | 代表标题 | 互动特征 |
|---------|---------|---------|
| **从零教程** | "Aider 完全指南：从零开始，用AI把编程效率提升4倍" | 高收藏（实用性强） |
| **对比安利** | "安利一个媲美Cursor的AI编程助手Aider" | 高点赞（踩热点） |
| **工具横评** | "AI编程助手工具链2026：Devin、SWE-agent与Aider" | 高评论（引发讨论） |
| **场景聚焦** | "终端福音，AI终端编程助手Aider" | 中等互动（垂直受众） |

### 1.2 Aider 的中文社区卖点（用户自发提炼）

1. **开源** — 反复出现，是核心信任建立点
2. **多 LLM 支持** — GPT-4/Claude/DeepSeek，"不绑死一家"
3. **Git 原生集成** — 自动 commit + diff，开发者刚需
4. **终端原生** — "不绑 IDE"，轻量
5. **完全免费** — 对比 Cursor 的付费模式

### 1.3 对 SelfPlay 的映射

| Aider 中文卖点 | SelfPlay 对应 | SelfPlay 额外优势 |
|---------------|-------------|-----------------|
| 开源 | 开源 MIT | 同 |
| 多 LLM 支持 | SDK-neutral（Mock/Claude/Codex） | 有 mock 零依赖演示 |
| 终端原生 | CLI-first | 同 |
| 免费对比 Cursor | 完全免费 | 同 |
| **无** | **自进化（Agent 修改自己的指令）** | **独有差异化** |

**关键发现**：Aider 在中文社区没有"自进化"叙事。SelfPlay 的"Agent 自己修改自己的规则"是全新品类。

## 2. Cursor 在 V2EX 的讨论模式

### 2.1 讨论主题分布

| 主题 | 情绪倾向 | 典型回复 |
|------|---------|---------|
| **Cursor 很强大** | 正面 | "第一次用就被震撼了" |
| **Cursor 使用心得/技巧** | 正面 | "rules 约束很重要"、"不同模型各有所长" |
| **Cursor 质量下滑** | **负面** | "最近版本不如以前了"、"产生看似正确实则错误的代码" |
| **Cursor 太贵** | **负面** | "500 个 request 额度烧完也没搞定" |
| **Cursor vs Claude Code** | 中性/对比 | 用户分裂为两个阵营 |
| **Cursor vs 免费工具** | 偏负面 | "开始找替代方案了" |

### 2.2 V2EX 用户痛点（与 user-feedback-insights.md 交叉验证）

| 痛点 | V2EX 表达 | 频率 |
|------|----------|------|
| "差一点就对了" | "AI 生成 90% 正确，最后 10% 修复更难" | 高频 |
| 质量退化 | "新版本不如以前" | 高频 |
| 费用敏感 | "500 request 烧完" | 中频 |
| 需要人工把控 | "框架需要人来把关" | 高频 |
| 横向对比需求 | "目前好用的 AI 编程助手是什么" | 高频 |

### 2.3 对 SelfPlay 的启示

1. **V2EX 用户对 Cursor 有"爱恨交织"情绪** — 爱其效率，恨其质量波动和价格
2. **"质量退化"是高频抱怨** — SelfPlay 的自进化是天然反叙事（越用越好，不是越用越差）
3. **费用敏感** — SelfPlay 开源免费 + mock 零 API key，直接回应
4. **"谁来把控"焦虑** — SelfPlay 的 OEDM 闭环提供了一种自动把控机制

## 3. 掘金高互动内容模式

### 3.1 最高互动类型排名

| 排名 | 类型 | 互动量 | 原因 |
|------|------|--------|------|
| 1 | **工具横评**（Cursor vs X vs Y） | 最高 | 用户需要决策帮助，评论区引发站队 |
| 2 | **实战教程 + 附源码** | 高 | 可复现，收藏价值高 |
| 3 | **"从零到一"教程** | 高 | 掘金用户偏好学习型内容 |
| 4 | **免费替代方案** | 高 | 价格敏感群体大 |
| 5 | **新品发布/发现** | 中等 | 需要独特卖点才能出圈 |

### 3.2 掘金 SelfPlay 最优内容策略

**不是"新品发布"，而是"从零到一教程 + 横评对比"**

推荐标题方向：
- ~~"SelfPlay：一个能自我进化的 AI Agent"~~（发布体，互动低）
- ✅ **"我让 AI Agent 自己修改自己的指令，结果..."**（悬念体）
- ✅ **"2026 年 5 个自进化 AI Agent 框架对比"**（横评体）
- ✅ **"从零搭建一个能自我进化的 AI Agent — SelfPlay 实战"**（教程体）

## 4. SelfPlay 中文发布策略调优

基于竞品数据，对 `research/chinese-community-launch-copy.md` 的调优建议：

### 4.1 V2EX 调优

| 原文案 | 调优后 | 原因 |
|--------|--------|------|
| 标题 `[开源] SelfPlay — AI Agent` | `[开源] SelfPlay — 让 AI Agent 自己改自己的指令，越用越聪明` | 加"自己改自己的指令"引发好奇 |
| 正文强调理论 | 弱化理论，加"Cursor 用户痛点"对照 | V2EX 用户已有 Cursor 使用经验 |
| — | 新增：直接对比 Cursor 的质量退化 vs SelfPlay 的持续进化 | 踩热点但不说坏话 |

### 4.2 掘金调优

| 原文案 | 调优后 | 原因 |
|--------|--------|------|
| "从零打造" 标题 | 保持，但加"附完整源码 + 可运行 demo" | 横评/教程是最高互动类型 |
| 正文最后问 3 个问题 | 改为"你最想让 AI Agent 学会什么？" | 更开放的讨论引导 |
| — | 新增：对比 aider（终端原生）+ Cursor（质量退化）| 横评体流量最大 |

### 4.3 即刻调优

即刻无需大改——情感钩子 "我做了一个会自己进化的 AI 🧬" 已经是最优解。
可补充：截图中加入 Cursor 负面评论的对照（"用户抱怨 Cursor 越用越差，SelfPlay 越用越聪明"）。
